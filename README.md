## Описание проекта

Проект содержит три варианта бота:

| Файл | Описание |
|------|----------|
| **bot_full_memory.py** | Основной бот: короткая + длинная память (рекомендуется) |
| bot_short_memory.py | Только короткая память (обычный чат-бот) |
| bot_long_memory.py | Только длинная память (бот-архивариус по документам) |

**Основной бот** (`bot_full_memory.py`) объединяет обе памяти:
- **Короткая память** — последние 10 сообщений диалога для связности разговора
- **Длинная память** — загруженные документы (PDF, TXT, DOCX), векторный поиск в ChromaDB

## Что нужно знать перед началом

### Необходимые аккаунты и ключи

1. **Telegram** — учётная запись в мессенджере
2. **Токен бота** — получается бесплатно у [@BotFather](https://t.me/BotFather) в Telegram
3. **API-ключ OpenAI** — получается на [platform.openai.com](https://platform.openai.com) (есть платные тарифы)

### Технические требования

- **Python 3.11 или 3.12** (3.14 не поддерживается — ChromaDB несовместим)
- Около 500 МБ свободного места для зависимостей
- Стабильный интернет (для OpenAI может потребоваться VPN в некоторых регионах)

### Важно про Python

Если у вас установлен Python 3.14, бот не запустится. Нужно установить Python 3.12 и запускать команды с указанием версии: `py -3.12 bot_full_memory.py`

## Логика работы бота

### Длинная память (документы)

1. Пользователь отправляет файл (PDF, TXT или DOCX)
2. Бот скачивает документ и извлекает текст
3. Текст разбивается на части (chunks) ≈500 символов с перекрытием 50
4. Чанки преобразуются в эмбеддинги (OpenAI `text-embedding-3-small`) и сохраняются в ChromaDB (папка `./memory`)
5. У каждого пользователя своя коллекция в векторной базе

### Короткая память (история диалога)

1. Последние 10 сообщений (вопросы и ответы) хранятся в оперативной памяти
2. При каждом запросе история передаётся модели OpenAI для учёта контекста разговора

### Генерация ответа на текстовое сообщение

1. **Поиск в длинной памяти** — семантический поиск релевантных фрагментов документов по вопросу (топ-5)
2. **Сборка промпта** — системная инструкция + контекст документов (если есть) + история диалога + текущее сообщение
3. **Ответ OpenAI** — модель отвечает, опираясь на документы и историю
4. **Обновление короткой памяти** — сообщение пользователя и ответ бота добавляются в историю

### Поведение

- **Без документов** — обычный диалог с учётом истории
- **С документами** — ответы по содержимому загруженных файлов + учёт истории
- **Только загрузка файла** — документ обрабатывается и индексируется

## Подробная инструкция по запуску

### Шаг 0: Получение токена Telegram-бота

1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Введите имя бота (например: «Мой помощник»)
4. Введите username бота (должен заканчиваться на `bot`, например: `my_helper_bot`)
5. BotFather пришлёт токен вида `7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
6. **Сохраните токен** — он понадобится для файла `.env`

### Шаг 1: Получение API-ключа OpenAI

1. Зарегистрируйтесь на [platform.openai.com](https://platform.openai.com)
2. Перейдите в раздел [API Keys](https://platform.openai.com/api-keys)
3. Нажмите «Create new secret key»
4. Скопируйте ключ (он начинается с `sk-`) и сохраните — повторно его показать нельзя
5. Убедитесь, что на счёте есть средства (OpenAI — платный сервис)

> **Важно:** в некоторых странах OpenAI недоступен напрямую. В этом случае потребуется VPN.

### Шаг 2: Установка Python

Если Python не установлен:

1. Скачайте **Python 3.12** с [python.org/downloads](https://www.python.org/downloads/)
2. При установке **обязательно** отметьте «Add python.exe to PATH»
3. Проверьте: откройте PowerShell и выполните `py --list` — должна быть версия 3.12

### Шаг 3: Скачивание проекта

Если проект ещё не на компьютере:

```bash
git clone https://github.com/for9653960484/owl_memory_bot.git
cd owl_memory_bot
```

Или скачайте ZIP-архив с GitHub и распакуйте его, затем откройте папку в терминале.

### Шаг 4: Установка зависимостей

Откройте **PowerShell** (Windows) или **Terminal** (macOS/Linux) в папке проекта и выполните:

```bash
pip install -r requirements.txt
```

Если используете Python 3.12 явно (рекомендуется):

```bash
py -3.12 -m pip install -r requirements.txt
```

Дождитесь окончания установки. Ошибок быть не должно.

### Шаг 5: Создание файла .env

1. В корне проекта (рядом с `bot_full_memory.py`) создайте файл с именем `.env`
2. Откройте его в любом текстовом редакторе
3. Вставьте следующие строки, подставив свои значения:

```env
BOT_TOKEN=7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

4. Сохраните файл (кодировка UTF-8)
5. **Не публикуйте файл `.env`** в интернете — в нём секретные ключи. Добавьте `.env` в `.gitignore`, если используете Git.

### Шаг 6: Запуск бота

В терминале (папка проекта) выполните:

```bash
py -3.12 bot_full_memory.py
```

Или, если Python 3.12 — версия по умолчанию:

```bash
py bot_full_memory.py
```

Успешный запуск выглядит так:

```
Бот запущен. Короткая + длинная память активны.
Загружайте документы и общайтесь.
```

> **Примечание:** при запуске бота может потребоваться подключение VPN (если API OpenAI недоступен в вашем регионе).

### Альтернатива: виртуальное окружение (для опытных)

Чтобы изолировать зависимости проекта:

```bash
py -3.12 -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
py bot_full_memory.py
```

### Запуск других вариантов бота

```bash
# Только короткая память (без документов)

# Только длинная память (только документы, без истории диалога)
```

## Как пользоваться ботом

1. **Найдите своего бота** в Telegram по username (который вы задали в BotFather)
2. **Нажмите «Start»** или отправьте `/start`
3. **Напишите сообщение** — бот ответит как обычный собеседник
4. **Загрузите документ** (PDF, TXT или DOCX) — отправьте файл как вложение. Бот обработает его и сообщит о готовности
5. **Задайте вопрос по документу** — например: «О чём этот документ?», «Какие основные выводы?»
6. **Продолжайте диалог** — бот помнит последние 10 сообщений и контекст разговора

## Решение проблем

### Ошибка: `ConfigError: unable to infer type for attribute "chroma_server_nofile"`

**Причина:** используется Python 3.14, ChromaDB его не поддерживает.

**Решение:** установите Python 3.12 и запускайте: `py -3.12 bot_full_memory.py`

### Ошибка: `ModuleNotFoundError: No module named 'aiogram'`

**Причина:** зависимости установлены для другой версии Python.

**Решение:** установите для нужной версии: `py -3.12 -m pip install -r requirements.txt`

### Ошибка: `Error code: 403 - Country, region, or territory not supported`

**Причина:** OpenAI API недоступен в вашем регионе.

**Решение:** включите VPN (сервер в США или Европе) и запустите бота снова.

### Ошибка: `Задайте BOT_TOKEN в .env или переменных окружения`

**Причина:** файл `.env` не найден или переменные не заданы.

**Решение:** убедитесь, что файл `.env` лежит в той же папке, что и `bot_full_memory.py`, и содержит строки `BOT_TOKEN=...` и `OPENAI_API_KEY=...` без лишних пробелов и кавычек.

### Бот не отвечает в Telegram

- Проверьте, что бот запущен (в консоли нет ошибок)
- Убедитесь, что токен в `.env` правильный
- Убедитесь, что интернет работает и OpenAI API доступен (при необходимости — через VPN)

### Документ не обрабатывается

- Поддерживаются только форматы: PDF, TXT, DOCX
- Файл должен содержать текст (скан без OCR не подойдёт)
- Размер файла не должен быть слишком большим (рекомендуется до нескольких МБ)

## Часто задаваемые вопросы

**Q: Бот бесплатный?**  
A: Сам бот — да. Но API OpenAI платный: списываются средства за запросы к модели и эмбеддингам.

**Q: Где хранятся данные?**  
A: Документы и эмбеддинги — в папке `./memory` на вашем компьютере. История диалога — в оперативной памяти и очищается при перезапуске бота.

**Q: Можно ли использовать на другом компьютере?**  
A: Да. Скопируйте папку проекта и файл `.env`. Папка `memory` при желании переносится вместе с проектом.

**Q: Как остановить бота?**  
A: Нажмите `Ctrl+C` в терминале, где он запущен.
